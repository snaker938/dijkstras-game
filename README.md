# Dijkstra's Game

Dijkstra's Game is a retro strategy puzzle about shortest paths, wall placement,
and algorithmic thinking. A missile is moving through a grid with one rule: it
always takes the shortest available path from the start node to the target. Your
job is not to block the missile completely. Your job is to keep a path open, but
make that path too long for the missile to survive.

The game includes a campaign, a sandbox builder, visual pathfinding animations,
custom grid saving, and a solver that can search for safe wall placements.

## Gameplay

Each level is a graph puzzle disguised as a tactical defence grid.

- The green node is the missile start.
- The red node is the target.
- Empty nodes are open space.
- Wall nodes are blocked space.
- Permanent walls are fixed obstacles.
- Random walls are generated by the random-wall power-up.

The key value is the **End Distance**. If the missile reaches the end node before
travelling farther than that distance, the mission fails. If the shortest valid
route is longer than the end distance, the missile self-destructs before impact.

```text
Win:     a path exists and d(start, end) > End Distance
Lose:    a path exists and d(start, end) <= End Distance
Invalid: no path exists
```

The invalid case matters. If every route is blocked, the enemy detects that the
navigation system has been compromised and patches the backdoor.

## Dijkstra's Algorithm

The grid is modelled as a graph:

```text
G = (V, E)
```

`V` is the set of open grid nodes. `E` is the set of allowed up, down, left, and
right moves between neighbouring open nodes. In this game, every move has unit
weight:

```text
w(u, v) = 1
```

Dijkstra's algorithm finds the shortest distance from a start node `s` to every
reachable node:

```text
dist[s] = 0
dist[v] = infinity for every v != s
```

It repeatedly chooses the unvisited node with the smallest known distance and
relaxes its neighbours:

```text
dist[v] = min(dist[v], dist[u] + w(u, v))
```

Because every edge has non-negative weight, once a node is chosen as the closest
unvisited node, its distance is final. In the game animation, this appears as a
wave expanding outward from the start node in distance order.

When the algorithm reaches the end node, it has found the exact shortest route
the missile will take.

## The Wall Puzzle

A campaign level gives you a wall limit `k` and an end distance `D`. In sandbox,
`k` is the remaining number of placeable walls after existing editable walls are
counted. You choose a set of new walls `W`:

```text
|W| <= k
```

Those walls remove nodes from the graph, producing a modified graph `G - W`. The
goal is:

```text
d_{G-W}(start, end) is finite
d_{G-W}(start, end) > D
```

That makes the puzzle a shortest-path interdiction problem: you are not looking
for the shortest route yourself, but for a small set of blocked nodes that makes
the missile's shortest route long enough to fail.

## Solver Overview

The sandbox solver searches for wall placements that satisfy the same win
condition as the campaign. It treats current walls, permanent walls, start and
end positions, the wall limit, and the end distance as solver input.

For every candidate solution, the solver verifies:

```text
path exists
walls used <= wall limit
shortest path length > end distance
```

The UI shows solver work as a blue wall overlay. You can inspect multiple safe
path solutions, preview them, hide the overlay, or implement a selected solution
as real walls. The solver runs through a worker so the interface can continue to
show progress while search is happening.

## Learning Mode

The home screen includes a Learning menu with a compact explanation of the grid,
Dijkstra's algorithm, the win condition, and the solver. The first campaign level
also includes a clearer tutorial that introduces the algorithm through the actual
rules of the game.

## Run

```powershell
npm install
npm start
```

Open `http://127.0.0.1:3000/`.

## Desktop App

```powershell
npm install
npm run desktop
```

`npm run desktop` builds the Vite app into `build/` and launches it in Electron.

If Electron reports that its binary did not install correctly on this Windows
machine, run:

```powershell
$env:NODE_OPTIONS='--use-system-ca'
npx install-electron --no
```

## Development Electron Window

```powershell
npm install
npm run dev
```

This starts the Vite dev server on port `3000` and opens Electron against it.
